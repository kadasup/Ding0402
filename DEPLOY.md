# 🚀 「自由543」發佈建議與架構指南

這套系統目前已經發展得非常完整，建議採用以下架構發佈上網，不僅**維護成本為 0**，且效能非常穩定。

## 🏗️ 建議架構

- **前端 (Web App)**: 使用 **Vercel** 託管。
  - **優點**：與 GitHub 無縫整合，每次您推代碼 (git push) 就會自動部署更新。
  - **費用**：個人版完全自費（Free）。
- **後端 (API/Database)**: **Google Apps Script (GAS)** + **Google Sheets**。
  - **優點**：免去租借資料庫的錢，所有數據都存在您的試算表，方便管理員直接開啟查看（例如導出 Excel）。
  - **費用**：完全免費。

---

## 🛠️ 分步發佈指南

### 第一步：後端連結 (Google Apps Script)
1. 在 Google 試算表中點擊 `工具` -> `指令碼編輯器 (Apps Script)`。
2. 點擊右上角 **部署** -> **新部署**。
3. 類型選擇：**網頁應用程式**。
4. **執行身分**：您的個人帳戶。
5. **誰可以存取**：**所有人 (Anyone)**。
   - *注意：這對公眾點餐非常重要，否則其他人會遇到 401 錯誤。*
6. 複製產生的 `Web App URL` (網址末尾為 `/exec`)。

### 第二步：設定前端環境變數
在專案根目錄（與 `package.json` 同級）建立 `.env` 檔案：
```env
VITE_GAS_URL=您的_GAS_網址_末尾是_exec
VITE_AZURE_API_KEY=AI辨識金鑰(若無可不填)
```

### 第三步：正式發表 (推薦 Vercel)
1. 前往 [Vercel](https://vercel.com/) 註冊並關聯您的 GitHub 帳號。
2. 點擊 **Add New Project**，選擇此專案 (`Ding0402`)。
3. 在部署設定的 `Environment Variables` 中，手動填入上述的 `VITE_GAS_URL`。
4. 點擊 **Deploy**。

大約 60 秒後，您就會得到一個專屬的網址（例如：`https://ding-543.vercel.app/`），點擊即可上網點餐！
